## Linked List Assignment

### Part 4: Concept Questions

**1. What is the role of a class in a linked list?**
A class serves as a template. We use a Node class to define each element 
The entire structure will be implemented using a LinkedList class and a pointer to the next element (data + pointer to next). 
Data structures and operations such as append and search.

**2. What is the difference between a node and a linked list?**
A node is one data point, including one value and a pointer to the next node. 
A linked list is the whole chain of connected nodes that is controlled by a head pointer.

**3. Why do we use None in next?**
The end of the list is indicated by the word None. It's the default value for a new node before it's connected, and signals that there is no following node.

**4. How is a linked list different from a Python list?**
A list is a Python data Structure that is used to store a lot of items in continuous memory and can be accessed in an instant by indexing it. 
A linked list is a list of nodes that are connected by pointers (no index access), but faster insertions at the front, without moving any elements.

**5. Why is OOP useful for data structures?**
OOP groups data and behavior together in clean, reusable classes. Each class clearly defined responsibility, making it easier to read, maintain, and extend code.
